EN IMAGE FETCHING WORKFLOW
==========================

How to find and download English card images from Serebii for a new JP set.
Written after the md/ns/nz image collection session (2026-05-21).


OVERVIEW
--------
JP sets don't always map 1:1 to EN sets. There are three cases:

  CASE A: Regular set (e.g. ns, nz)
    - One JP set maps cleanly to one EN set
    - Card numbers match 1:1 for the main cards
    - EN set may have a few extra EN-exclusive cards at the end

  CASE B: High Class set (e.g. md / Mega Dream ex)
    - One JP set is a premium compilation that reprints cards from many previous JP sets
    - EN images are scattered across many different EN sets
    - Requires card-by-card matching

  CASE C: No EN equivalent
    - Card is JP-only, no EN image exists
    - Leave imageEn: null in the card JSON


STEP 1 — IDENTIFY THE EN SET(S)
---------------------------------
1. Go to https://serebii.net/card/{setname}/ for the JP set
   (e.g. https://serebii.net/card/ninjaspinner/)

2. Fetch the first ~30 card names from the JP set page.

3. Go to https://serebii.net/card/english.shtml for a list of EN sets
   with release dates and card counts.

4. Fetch the card list for candidate EN sets and compare the first
   10-20 card names. If they match in the same order → that's your EN set.

5. For regular sets: the JP and EN card numbers match 1:1 for the main cards.
   EN sets often add a few extra trainers/energies at the end (e.g. PO has 88
   cards vs nz's 80 — the extra 8 are EN-only). Download JP-count cards only.

CONFIRMED REGULAR SET MAPPINGS (Mega Evolution era):
  ns (Ninja Spinner, 83 JP)    → chaosrising   (86 EN)
  nz (Nihil Zero, 80 JP)       → perfectorder  (88 EN)


STEP 2 — IMAGE URL FORMAT
--------------------------
Serebii card image URLs follow this pattern:
  https://serebii.net/card/{setname}/{n}.jpg

Where {setname} is the Serebii URL slug (lowercase, no spaces):
  ninjaspinner, nihilzero, megadreamex
  chaosrising, perfectorder, ascendedheroes
  destinedrivals, whiteflare, blackbolt
  megaevolution, phantasmalflames, surgingsparks
  twilightmasquerade, shroudedfable, stellarcrown
  journeytogether, etc.

And {n} is the card number WITHOUT zero-padding (1.jpg, not 001.jpg).

Verify the URL pattern by fetching https://serebii.net/card/{setname}/001.shtml
and checking the image src in the page content.


STEP 3 — DOWNLOAD WITH POWERSHELL
------------------------------------
Use Invoke-WebRequest with browser headers to avoid blocks.
Always add -Encoding UTF8 for any text operations (not needed for images).

TEMPLATE — Regular set (1:1 mapping):

  $headers = @{
      'User-Agent' = 'Mozilla/5.0 (Windows NT 10.0; Win64; x64) AppleWebKit/537.36'
      'Referer'    = 'https://serebii.net/'
  }
  $destDir = "images\sets\{setIdJa}\en"
  for ($i = 1; $i -le {jpCardCount}; $i++) {
      Invoke-WebRequest -Uri "https://serebii.net/card/{enSetName}/$i.jpg" `
          -Headers $headers -OutFile "$destDir\$i.jpg"
      Start-Sleep -Milliseconds 150
  }

  Save as: images/sets/{setIdJa}/en/{jp_card_number}.jpg
  The filename = JP card number (not EN card number).


STEP 4 — HIGH CLASS SETS: BUILDING THE MAPPING TABLE
------------------------------------------------------
For High Class sets, you need to map each JP card → EN set + EN card number.

Process:
  1. Fetch the FULL JP card list from Serebii (all normal cards, 1 to N)
  2. Fetch full card lists for ALL candidate EN sets
  3. Match cards by name (same Pokémon name = same card, usually)
  4. Build a hashtable: $mapping = @{ jpNum = @(enSetName, enNum); ... }

TEMPLATE — High Class set download:

  $mdToEN = @{
      1  = @("destinedrivals", 1)
      4  = @("ascendedheroes", 11)
      # ... etc
  }
  foreach ($jpNum in ($mdToEN.Keys | Sort-Object)) {
      $enSet = $mdToEN[$jpNum][0]
      $enNum = $mdToEN[$jpNum][1]
      Invoke-WebRequest -Uri "https://serebii.net/card/$enSet/$enNum.jpg" `
          -Headers $headers -OutFile "$destDir\$jpNum.jpg"
      Start-Sleep -Milliseconds 150
  }


STEP 5 — HIGH CLASS SET: WHICH EN SETS TO CHECK
-------------------------------------------------
Start with the most likely EN set (the one closest in time to the JP HC set).
Then expand outward to older sets for cards not found.

For the Mega Evolution era (2025-2026), the search order that worked for md:
  1. ascendedheroes   — direct EN partner (Jan 2026), covers ~70% of cards
  2. destinedrivals   — covers character cards (Ethan's, Cynthia's, TR cards)
  3. journeytogether  — covers Hop's cards + some items/stadiums
  4. surgingsparks    — covers some SV Pokémon ex (Ceruledge ex, Chien-Pao, etc.)
  5. twilightmasquerade — covers Teal Mask Ogerpon ex, Tatsugiri, Enhanced Hammer
  6. shroudedfable    — covers Duskull line
  7. stellarcrown     — covers Hoothoot/Noctowl, Area Zero Underdepths
  8. whiteflare       — covers Unova/White: Reshiram ex, Emboar, Hilda, Ignition Energy
  9. blackbolt        — covers Unova/Black: Zekrom ex, Genesect ex
  10. megaevolution   — covers Mega Evolution Pokémon: Magnemite/Magneton, etc.
  11. phantasmalflames — covers some SV Pokémon: Charcadet, Duraludon

TIPS:
  - If a Pokémon has a clear "home" era (BW=BB/WF, SW/SH=JT, SV=SS/SC/TM/PF),
    start searching there.
  - Character cards (Ethan's, Cynthia's, Hop's, N's, TR cards) tend to cluster
    in DRI and JT.
  - Items and stadiums often appear in the same EN set as the Pokémon they
    support (e.g. Hop's Bag/Choice Band → JT alongside Hop's Pokémon).
  - Ignition Energy (Fire special energy with discard-then-hand effect) → WF 86
  - Secret rares in EN sets are alt arts of existing cards, not new cards —
    so if a card isn't in the normal range of an EN set, it won't be in that
    set's secret rares as a new card.

SAVING IMAGES
--------------
  Destination: public/images/sets/{setIdJa}/en/{jp_card_number}.jpg
  Filename = JP card number (NOT EN card number)
  This is how the JSON imageEn field references local images:
    "imageEn": "images/sets/{setIdJa}/en/{n}.jpg"

  Verify counts:
    (Get-ChildItem "images\sets\{setIdJa}\en").Count
