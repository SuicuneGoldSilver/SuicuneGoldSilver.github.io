BUG FIX — MODAL NOT SHOWING TEXT FOR jtp CARDS
================================================================================

PROBLEM:
  jtp cards have imageEn: null (no EN counterpart). In useModal-Bne0OhqI.js:
    - N = !!s.imageEn → false for jtp cards
    - EN button render condition: N && e.jsx("button", ...) → EN button never rendered
    - Cards opened in JP mode by default (correct), but textJa was null at the time

  After textJa was populated, modal correctly shows JP text. No JS change needed.
  The root cause was just missing data, not a code bug.

ATTEMPTED FIX (reverted):
  Tried changing EN button condition to show it when textEn exists even without imageEn.
  User clarified: jtp cards should ONLY show the JP button. Reverted to original condition.

FINAL STATE:
  - imageEn: null on all jtp cards → N = false → EN button never shown → JP-only display
  - This is correct behavior for this set

================================================================================
BUG FIX — SEARCH BLANK PAGE WHEN "ALL SETS" SELECTED
================================================================================

PROBLEM:
  When searching across all sets (no set filter), the page went completely blank.
  Root cause: filter function calls E(t.nameJa).includes(S) where E(a) = a.toLowerCase()
  This throws TypeError: Cannot read properties of null (reading 'toLowerCase')
  when nameJa is null.

INVESTIGATION:
  - Searched all 3443 index entries for null nameJa
  - Found exactly 1 card: entp-59 (Brilliant Blender, SSP 164) — EN-only ACE SPEC promo
    with no JP counterpart. Was added to entp.json without a Japanese name.

FIX:
  - Set nameJa = 'ブリリアントブレンダー' on entp-59 in data/cards/entp.json
  - Rebuilt index: 3443 entries, 0 null nameJa
  - Search across all sets now works without crashing

================================================================================
KEY LEARNINGS / NOTES
================================================================================

- Modal language toggle logic (useModal-Bne0OhqI.js):
    N = !!s.imageEn             → EN button only renders when imageEn is non-null
    enOnly = N && imageJa===imageEn  → true = same image used, shows EN-only
    Default open lang: imageEn && imageJa===imageEn ? "en" : "ja"
    → Cards with imageEn: null always open in JP mode

- Any card added to the data files MUST have nameJa populated. The search filter
  calls .toLowerCase() on nameJa directly without null-checking. A single null
  nameJa will crash the entire search component.

- BOM preservation: jtp.json and entp.json both start with '﻿'. Scripts must
  detect and preserve the BOM on write (detect with raw.startsWith('﻿')).
